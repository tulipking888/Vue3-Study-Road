# 七、角色字段设计


常见字段：


```ts
interface Role {


id:number


name:string


code:string


status:number


remark:string


}

```


---

解释：


## id


数据库主键。


---

## name


角色名称。


例如：

```
管理员

普通用户

客服

```


---

## code


角色编码。


例如：

```
ADMIN

USER

CUSTOMER_SERVICE

```


程序判断权限通常使用code。


---

## status


状态。


例如：

```
0 禁用

1 正常

```

# 二十三、常见问题


## 问题1

为什么角色不用直接写权限？


因为：

角色可以复用。


例如：

100个客服：

只绑定一个客服角色。


---

## 问题2

为什么需要code？


代码判断：

不要依赖中文。


例如：


错误：

```
if(role=="管理员")

```


正确：

```
if(role=="ADMIN")

```


---

## 问题3

为什么菜单也属于权限？


因为：

菜单决定：

用户能看到什么。


按钮决定：

用户能操作什么。
